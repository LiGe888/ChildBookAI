# 互动与音频技术方案

## 结论

可以实现。建议第一阶段做成本地静态网页：打开网页后选择课程，点击漫画画面播放英文，当前句高亮，孩子可以点选答案，也可以录音跟读。这样不需要服务器，也不依赖复杂后端。

| 能力 | 当前状态 | 用途 |
| --- | --- | --- |
| Node.js | 已发现 `/opt/homebrew/bin/node` | 后续可做前端开发、批量生成页面 |

## 第一阶段 MVP

不安装任何新依赖也能做。

- `app/static/index.html`：课程入口
- `app/static/styles.css`：儿童绘本界面样式
- `app/static/app.js`：点击播放、文字高亮、答题、录音
- `app/static/lessons.json`：课程数据
- `assets/images/`：图片
- `assets/audio/`：音频

## 音频方案

### 方案 A：浏览器朗读

使用 Web Speech API。优点是快，不需要先生成音频文件。缺点是不同浏览器音色不同，角色音色控制弱。

适合：早期原型。

### 方案 B：macOS say 生成本地音频

使用系统 `say` 命令逐句生成 `.aiff` 文件。

示例：

```bash
say -v Samantha -r 130 -o assets/audio/S1-W01-L01-01.aiff "Hello!"
```

优点是稳定、可离线。缺点是 `.aiff` 文件较大，若要转 MP3 需要 `ffmpeg`。

适合：本地离线版本。

### 方案 C：真人录音或 TTS API

优点是情绪自然、角色感强。缺点是制作成本高，管理文件较多。

适合：内容稳定后再做。

## 点击互动

建议先做 5 类互动：

- 点画面播放当前句。
- 点角色播放角色名。
- 点目标物品回答问题。
- 点按钮录音跟读。
- 点复习题得到正确/再试一次反馈。

## 动图方案

第一阶段优先用 CSS 动效，不急着做 GIF。

- 星星闪烁：CSS opacity/scale
- 角色跳起：CSS transform
- 物品出现：CSS transition
- 翻页：CSS class 切换

后续如果要真正动图：

- GIF/APNG：需要图片序列和转换工具，建议补 ImageMagick 或 ffmpeg。
- MP4/WebM：适合较复杂动画，建议补 ffmpeg。
- Lottie：适合矢量动画，但需要设计或生成 Lottie 文件。

## 跟读录音

浏览器可以用 MediaRecorder 录音，默认先只保存在本地页面内，不上传。

第一阶段不做自动评分，只做：

- 点击录音。
- 点击停止。
- 回放自己的声音。
- 家长手动判断是否愿意跟读。

## 进度记录

第一阶段用 `localStorage` 记录：

- 已完成课程
- 每课听了几遍
- 跟读次数
- 答题正确数

后续如需多设备同步，再考虑后端或云端。

## 风险与处理

| 风险 | 处理 |
| --- | --- |
| 4 岁孩子注意力短 | 每课 8 分钟左右，画面少，动作多 |
| TTS 音色不够可爱 | 原型阶段可接受，稳定后换真人或 API |
| 图片生成风格不一致 | 先固定角色设定和提示词，再批量生成 |
| 动效制作成本高 | 先用 CSS 微动效，后续再做复杂动画 |
| 版权风险 | 坚持原创角色，不使用知名 IP 形象 |
